Japan Trip — Encrypt Builder

Generates a password-gated index.html from your plaintext source.html. Runs entirely in your browser — nothing leaves this page.

How to use

  1. Pick a strong passphrase (4+ words is best).
  2. Drop or pick your source.html below.
  3. Click Encrypt — your browser downloads index.html.
  4. Commit and push index.html to your GitHub repo. Do NOT commit source.html.
  5. To rotate the password later: edit source.html, come back here, repeat.

Step 3 — Build

Security notes — read once

Encryption. AES-256-GCM, key derived via PBKDF2-SHA256 with 200,000 iterations from your passphrase. Random salt + IV per build. The password itself is never stored anywhere — only the AES key derived from it is cached in sessionStorage after a successful unlock, and that cache clears when the browser tab closes.

The encrypted blob is permanently public. Once you push index.html to GitHub, anyone can download the ciphertext and brute-force it offline. PBKDF2 with 200k iterations slows this down significantly, but the only real defense is a strong passphrase. Use 4+ random words.

Rotating the password creates a new ciphertext but does not invalidate copies of the old one. Anyone who downloaded the file before rotation can keep attacking the old version. Rotate if you suspect leakage, but don't expect rotation alone to protect already-leaked content.

Git history. If you ever committed source.html (the plaintext) by mistake, scrub it from git history — .gitignore only blocks future commits, not past ones. Use git filter-repo or BFG Repo-Cleaner.

Local previews. Open source.html directly to preview without password. Open the encrypted index.html to test the unlock flow.
